Setup Guide

Activate Plaid Account Aggregation

Link 12,000+ U.S. financial institutions. Balances auto-populate into the plan, refreshed on demand. 15-minute one-time setup; then clients connect accounts in-app.

Checking Plaid endpoint status…
1

Create a Plaid account

~ 5 minutes
  1. Go to dashboard.plaid.com/signup and sign up with your firm's email.
  2. Complete the business-info form (firm name, address, use case = "Financial Planning").
  3. After approval (usually instant for financial planning), you'll land in the dashboard.
  4. Navigate to Team Settings → Keys in the left sidebar.
  5. Copy your client_id (shown plaintext) and Sandbox secret (click "Reveal").
Sign up for Plaid →
💰 Cost: Plaid Sandbox is free and unlimited. Development and Production use an account-based model: ~$0.30/linked account/month for basic balances. For 50 linked clients × 3 institutions each = ~$45/month in production. Sandbox first, graduate to production when ready.
2

Add Plaid environment variables to Vercel

~ 3 minutes

In your Vercel project → Settings → Environment Variables, add three variables (apply to Production, Preview, Development):

PLAID_CLIENT_ID

Value: your client_id from Plaid dashboard

PLAID_SECRET

Value: your Sandbox secret (switch to Production secret when going live)

PLAID_ENV

Value: sandbox (start here) or production (when live)

Open Vercel Dashboard →
3

Add Supabase service role key

~ 2 minutes

The Plaid exchange endpoint writes to a private table in Supabase (plaid_items). This requires the service-role key, not the anon key.

  1. Open your Supabase project → Project Settings → API
  2. Under Project API keys, copy the service_role key (starts with eyJ...)
  3. In Vercel → Environment Variables, add:
SUPABASE_SERVICE_ROLE_KEY
SUPABASE_URL

Value: your Supabase project URL (e.g., https://xxxxx.supabase.co)

⚠️ Security note: the service-role key bypasses row-level security. Never expose it to the client — it must only live in Vercel env vars and be used by serverless functions. Never paste it into index.html or config.js.
4

Run the Supabase migration

~ 1 minute

After running you should see "Success. No rows returned". Verify in Table Editor that plaid_items appears.

5

Redeploy on Vercel

~ 1 minute

Env vars only activate after a fresh deploy. In Vercel → Deployments → top row → ⋯ → Redeploy (check "Use existing Build Cache"). Wait for the green ✓.

6

Test the connection

~ 30 seconds

Clicks /api/plaid/link-token to verify the key is configured. Doesn't actually link an account — just confirms the handshake works.

Once the test passes, open the main app, load a plan, and the "🔗 Connect Accounts" button in the builder will launch Plaid Link.
