Custom domain: www.pylefinancialservices.com Required

~ 10 minutes + DNS propagation

Gets rid of the pyle-planning.vercel.app URL. Clients see a real domain that matches your firm.

  1. Open Vercel → your project → Settings → Domains.
  2. Click Add Domain and type www.pylefinancialservices.com.
  3. Vercel shows you DNS records to add at your domain registrar (GoDaddy, Cloudflare, etc.).
  4. Add the records at your registrar. Usually a CNAME from www → cname.vercel-dns.com.
  5. Also add the root redirect: Vercel will walk you through making pylefinancialservices.com redirect to www.pylefinancialservices.com.
  6. Wait 5-30 minutes for DNS propagation. Vercel auto-provisions the SSL certificate.
Open Vercel Domains
After the domain is live: update Supabase → Auth → URL Configuration so the Site URL is https://www.pylefinancialservices.com. Otherwise magic links will still redirect to the vercel.app URL.

Sentry error monitoring Free tier

~ 5 minutes

Catches JavaScript errors in the field before clients report them. If something breaks in Monte Carlo for a specific client's inputs, you'll see it in Sentry before they see it on screen.

  1. Create a Sentry account (free tier: 5k errors/month — plenty).
  2. Create a new project → pick JavaScript → Browser.
  3. Copy the DSN (looks like https://abc123@o12345.ingest.sentry.io/67890).
  4. In your project folder, open config.js and paste the DSN into the SENTRY_DSN field.
  5. Push the change with the desktop button. Sentry activates automatically on next page load.
Create Sentry account
Privacy note: I've already wired the Sentry SDK to scrub dollar amounts and SSN-like patterns out of error reports before they leave the browser. Client PII never reaches Sentry.
3

Email notifications via Resend Free tier

~ 15 minutes

Automatic emails when a plan is updated, a document is uploaded, or a review is posted. Free tier is 3,000 emails/month — well beyond what a solo practice needs.

  1. Create a Resend account.
  2. Add your domain (pylefinancialservices.com) under Domains. Resend gives you DNS records — add them at your registrar. This takes ~15 minutes to verify.
  3. Create an API key. Copy it.
  4. Add these three env vars to Vercel (Settings → Environment Variables):
    RESEND_API_KEY
    Value: paste the API key from step 3.
    NOTIFY_FROM
    Value: Pyle Planning <notify@pylefinancialservices.com>
    NOTIFY_TEAM
    Value: Admin@PFS4u.com
    APP_URL
    Value: https://www.pylefinancialservices.com
  5. Trigger a redeploy so the env vars pick up.
Create Resend account Open Vercel Env Vars

Branded magic-link email Polish

~ 5 minutes

Replaces Supabase's default "Confirm your email" with a properly-branded email carrying your firm colors, tagline, and a large "Open My Plan" button.

  1. Open Supabase Auth → Email Templates → Magic Link.
  2. Click the button below to copy the pre-written HTML to your clipboard.
  3. Paste into the Supabase template body.
  4. Set the subject to: Sign in to your financial plan — Pyle Financial Services
  5. Click Save.
  6. (Optional) Do the same for the Invite user template using invite.html.
Open Supabase Templates
5

Verify automated backups are on Required

~ 2 minutes

You just upgraded to Supabase Pro — which means Point-in-Time Recovery (PITR) is available. Verify it's actually turned on. Without this, if anything ever corrupts the database, the best you can do is roll back to yesterday's backup.

  1. Open Supabase → Database → Backups.
  2. Confirm "Point-in-Time Recovery" is Enabled.
  3. If not, click Enable — it's included in your Pro plan.
Open Supabase Backups
Why this matters: PITR lets you restore to any moment in the last 7 days. If an advisor accidentally deletes a plan on Wednesday, you can roll back to Tuesday afternoon without losing the other work done that week.
